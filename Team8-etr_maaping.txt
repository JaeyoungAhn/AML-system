/*ER to Relational Mapping*/

1. Mapping of Regular Entity Types
(BANK, ACCOUNT, HOLDER, TRANSACTION, DNG_PERS, DNG_ACCT,COUNTRY_CREDIT)

- BANK의 ADDRESS 속성을 Simple attribute(STREET_ADDRESS, CITY, STATE, ZIP_CODE)로 세분화
- Primary Key: BANK_ID
- Super Key : BRANCH_NAME

- ACCOUNT
- Primary Key: BANK_ID
- Super Key : ACCOUNT_NUMBER

- HOLDER
- Primary Key: H_ID
- Super Key : PHONE_NUMBER

- TRANSACTION
- Primary Key : TXN_ID

- DNG_PERS
- Primary Key: DNG_ID

- DNG_ACCT
- Primary Key : ACCT_NO

- COUNTRY_CREDIT
- Primary Key : Name 


2. Maaping of Weak Entity Types
(DNG_HOLDER, DNG_TXN)

- DNG_HOLDER
- Primary Key : H_ID
- Foreign Key : HOLDER의 PK인 H_ID

-DNG_TXN (DDL 수정필요)
- Primary Key : TXN_ID
- Foreign Key : TRANSACTION의 PK인 TXN_ID

3. Mapping of Bianry 1:1 Relation Types
(NOTHING)

4. Mapping of Binary 1:N Relationship Types
a) S: ACCOUNT, T: HOLDER
- HOLDER의 PK인 H_ID를 ACCOUNT의 FK로 추가

b) S: TRANSACTION, T: ACCOUNT
- ACCOUNT의 PK인 ACCOUNT_NUMBER를 TRANSACTION의 FK로 추가

c) S: DNG_PERS, T: BANK
- BANK의 PK인 BANK_ID를 DNG_PERS의 FK로 추가

d) S: DNG_ACCT, T: BANK
- BANK의 PK인 BANK_ID를 DNG_ACCT의 FK로 추가

e) S: COUNTRY_CREDIT, T: BANK
- BANK의 PK인 BANK_ID를 COUNTRY_CREDIT의 FK로 추가

5. Mapping of Binary M:N Relationship Types
(NOTHING)

6. Mapping of Multivalues attributes.
(NOTHING)

7. Mapping of N-ary Relationship Types.
- R: HOLDER, BANK, TRANSACTION
새로운 U 생성 : INITIATION
각 R의 PK ---> U의 FK(H_ID, BANK_ID, TXN_ID)
